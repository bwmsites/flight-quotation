    Bilheteria Digital fornece API REST simples e completa para criação de sistema de gestão de eventos.

    Esta documentação fornece informações sobre pontos de extremidades disponíveis do endpoint e como interagir com eles.

    Informações importantes:

    • Deve ser enviado no path em todos os serviços (exceto login) o token de autenticação obtido no serviço de login;

    • Qualquer atributo da model a ser retornado pode ser usada como filtro na requisição, caso possua algum filtro diferenciado estará disponível na descrição do serviço;

    • Em cada serviço tem o link para a documentação interativa com a descrição de modelo do serviço;

    • No serviço de POST o id é desconsiderado, pois irá cadastrar os demais campos e então criar um id;

    • Nos serviços de PUT deve ser enviada a model preenchida com os dados corretos para o registro ser atualizado e filtrado pelo id enviado;

    • Todos campos de data são inteiros, no formato timestamp;

    • Todos a requisição e resposta acontece através de documentos json.

    Estrutura padrão para os verbos e ações

    Campo Descrição
    GET Busca registros
    POST Cadastra registros
    PUT Atualizar registros
    DELETE Desabilita\Exclui registros

    Estrutura padrão para respostas

    Campo Tipo Valor Descrição
    secesso Bolean Define se a requisição foi sucesso ou falha, caso seja falha mensagem virá com valor.
    mensagem Array Array de strings contendo mensagem de erro
    dados mixed Irá conter o valor da resposta. Mudará seu tipo conforme a requição de cada serviço.

    Proposta

    Endpoint proposta.

    Para filtros e crud em Proposta veja a Introdução

    A Proposta comercial dá inicio ao cadastro de eventos.

    Ao cadastrar uma proposta deve ser carregado todas as opções do endpoint ´proposta/tarifas´ e ´proposta/servico´ para permitir ao usuário editar seus valores.

    Se o usuário comercial preencher algum dos valores abaixo do valor padrão a proposta irá automaticamente para Aguardando Gerente até que um gerente aprove.

    Gerente é um usuario cuja seu id esteja presente no endpoin proposta/gerente

    O comercial cria a proposta que fica com o status Nova Proposta até ele decidir enviar para o produtor.

    Se o sistema verificar as taxas oferecidas são muito baixas a proposta é enviada para o gerente comercial com o status Aguardando Gerente que pode cancelar (Cancelada), pode aprovar e vai diretamente para o produtor (Avaliando) ou recusar e pedir nova análise pelo comercial (Recusada pelo Gerente).

    Se o sistema verificar que as taxas estão ok vai direto para o Produtor (Avaliando)

    Uma vez com o produtor (Avaliando) ele por pedir uma nova proposta (Recusada), cancelar de vez a negociação (Cancelada) ou aceitar os valores (Confirmada).

    Os status Recusada e Recusada pelo Gerente ficam disponíveis para o comercial reeditar essa proposta (Renegociando) até querer reenviar para o sistema entrar novamente no ciclo.

    O status Confirmada envia uma solicitação para DocYouSign para o produtor informar uma assinatura digital. Essa assinatura causa um notificação do site da DocYouSign de volta para a Bilheteria que gera automaticamente o Evento. O evento gerado ainda poderá ser editado pelo comercial que colocará todas as informações básicas como setores, ingressos, moedas e datas. Se o evento for marcado para ser configurado pela equipe técnica da BD eles receberão uma notificação no sistema, caso o contrário o Produtor receberá um e-mail que o seu evento esta disponível para configuração.

    Relatórios

    Os serviços de relatórios são somente leitura, eles servem os dados para listar os relatórios disponíveis, bem como os filtros e resultados.

    Listagem de Relatórios

    Para obter a lista de relatórios consulte o serviço com GET no endpoint relatorio.

    Irá retornar um array com os seguintes dados:

    Campo Tipo Descrição
    id integer Código interno
    relCapaId integer Campo que identifica um relatório pai.
    chave string Chave para identificar um relatório específico
    nome string Nome do relatório

    O campo relCapaId será utilizado para referenciar o id de um relatório pai para o relatório, para poder montar uma árvore de relatórios.

    Caso seja necessário buscar um relatório especifico em determinada parte do sistema pode ser utilizado o campo chave, fixando então um relatório sem precisar percorrer a lista.

    http://34.205.79.144/docs/#operations-Relatorio-relatorioDetalheGET

    Detalhes do Relatório

    Quando um relatório especifico for requisitado é necessário consumir o serviço com GET no endpoint relatorio\detalhe para obter os detalhes do relatório e filtros a serem apliados.

    Formato do filtro reservado:

    [
  {
    "Name": "name do filtro pré estabelecido", //valor retornado no serviço de detalhes
    "Value": mixed, // valor para aplicar no filtro.
  },
  ...
]

    Na requisição é possível informar filtros reservados que irão restringir os dados a serem exibidos nos filtros com valores pré estabelecidos.

    Filtros Reservados:

    Filtro Descrição
    EVENTO_ID Identificador de um evento

    Será retornado o array de filtros com os seguintes dados:

    Campo Tipo Descrição
    id integer Código interno
    relCapaId integer Campo que identifica o relatório a qual o filtro pertence
    tipo string Tipo de filtro que será aplicado
    valores string Valores pré definidos o filtro

    Formato de valores:

    [ 
  [value, label], 
  [value, label], 
  ... 
]

    O tipo de filtro pode ser:

    • Data: dois inputs text para filtro de datas;

    • Select: filtro do tipo <select> observando os dados do campo valore;

    • Valores: Os valores virão no formato ao lado;

    • Text: filtro com input text;

    • Number: filtro com input text validando somente numeros ou input number.

    operations-Relatorio-relatorioDetalheGET

    Resultado do Relatório

    Formato do filtro:

    [
  {
    "Name": "name do filtro", //valor retornado no serviço de detalhes
    "Value": ["valor"], // valor para aplicar no filtro, quando for data mandar duas posições.
  },
  ...
]

    Para obter mais o resultado do relatório é necessário consumir com GET o endpoint relatorio/resultado.

    Envie o filtro id correspondente a qual relatório está buscando e os filtros correspondentes ao reltório.

    Formato do agrupamento:

    [
  {
    "Label": "label do agrupamento", //valor retornado no serviço de detalhes
    "Value": "valor", // nome do agrupamento que será aplicado.
  },
  ...

    Envie o agrupamento.value correspondente para agrupar os dados por alguma informação disponível.
